1. クラス

設計図にあたるもの＝クラスを作成する
属性＝データ
振る舞い（操作）＝処理

まずは
- 属性の項目
- 振る舞いの項目
を列挙する（上流工程の設計段階で必要なことだろうな…）

2. クラスの構成要素
クラス
    フィールド
    メソッド

フィールド、メソッドをクラスのメンバという

3. フィールド
フィールド＝メンバ（メンバ変数といわれる）
→クラスの直下に記述する

変数のスコープ→Cとだいたい同じっぽい

4. メソッド
Cの関数と同じっぽい
staticは後述
基本つけなくてOK

e.g.,
public class SalesEmployee {
    // field
    String name; 
    int salesAmt; // 売り上げ金額

    // method
    int sell(int amt) { //　売り上げ処理
        return salesAmt += amt;
    }
}

5. クラス図
UML

クラス名
---------
フィールド→ フィールド名: データ型
---------
メソッド　→　メソッド名(引数名: 引数の型, ...): 戻り値の型
